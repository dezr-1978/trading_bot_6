# 📘 Data README

Цей документ описує формат даних, які використовує трейдинг‑платформа **Trading Bot FULL PRO**.  
Файли з історичними свічками повинні зберігатися в папці:

```
data/
```

---

## 📂 Формат CSV

Файл CSV **обов’язково** повинен містити такі колонки:

| Column      | Type    | Description |
|-------------|---------|-------------|
| timestamp   | int     | Unix time (мс або сек — обидва підтримуються) |
| open        | float   | Ціна відкриття свічки |
| high        | float   | Максимальна ціна |
| low         | float   | Мінімальна ціна |
| close       | float   | Ціна закриття |
| volume      | float   | Обсяг торгів |

---

## 📝 Приклад CSV (1 рядок)

```
timestamp,open,high,low,close,volume
1704067200000,43000.5,43120.0,42850.2,43080.1,123.45
```

Пояснення:

- `timestamp` — Unix time у мілісекундах (Binance формат)  
- `open` — ціна відкриття  
- `high` — максимум  
- `low` — мінімум  
- `close` — ціна закриття  
- `volume` — обсяг  

---

## ⏱ Підтримка timestamp у секундах і мілісекундах

Система автоматично визначає формат:

- якщо число > 10^12 → мілісекунди → буде конвертовано в секунди  
- якщо число < 10^12 → секунди → використовується напряму

## 🔧 Як підготувати CSV для Backtest

1. Завантажити історію (наприклад, з Binance):
   ```
   https://data.binance.vision/
   ```
2. Розпакувати ZIP
3. Перейменувати колонки під формат:
   ```
   timestamp,open,high,low,close,volume
   ```
4. Покласти файл у:
   ```
   data/BTCUSDT_15m.csv
   ```

## 📌 Важливі вимоги

- Дані повинні бути **чистими** (без пропусків у timestamp)
- Свічки повинні бути **посортовані за часом**
- Не повинно бути дублікатів timestamp
- Не повинно бути нульових значень у OHLC
